# gbtmkit: Engine-Agnostic Group-Based Trajectory Modelling

A generic, reproducible pipeline for group-based trajectory modelling
(GBTM) that follows the GRoLTS reporting checklist. Estimation is
delegated to interchangeable backends behind a single interface, so
binary and continuous outcomes are handled through one specification.

## Details

See the design document (`dev/DESIGN.md`) for the architecture, and the
`getting-started` vignette for a worked example on the synthetic
datasets
[sim_binary](https://fabregithub.github.io/gbtmkit/reference/sim_binary.md)
and
[sim_continuous](https://fabregithub.github.io/gbtmkit/reference/sim_continuous.md).
